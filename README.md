# Design System FAQ

Interactive FAQ for onboarding new product designers to the design system. Built with Next.js and distributed as self-contained platform binaries — designers run a single executable, no installs required.

## For designers

You need two things from the `dist/` folder:

1. The binary for your OS (see table below)
2. The `out/` folder — keep it in the same directory as the binary

| File | Platform |
|---|---|
| `designer-faq-mac-arm` | Mac with Apple Silicon (M1, M2, M3, M4) |
| `designer-faq-mac-intel` | Mac with Intel processor |
| `designer-faq-windows.exe` | Windows |
| `designer-faq-linux` | Linux |

**Mac/Linux:** Open Terminal, `cd` to the folder, then run:
```bash
./designer-faq-mac-arm   # or the variant for your Mac
```

> First time on Mac you may get a security prompt. Go to System Settings → Privacy & Security and click "Open Anyway".

**Windows:** Double-click `designer-faq-windows.exe`.

The browser opens automatically at `http://localhost:3000`.

## For developers

### Prerequisites

**Node.js** — to build the Next.js project.

**Bun** — to compile the platform binaries. Install it with:

```bash
# Mac / Linux
curl -fsSL https://bun.sh/install | bash

# Windows (PowerShell)
powershell -c "irm bun.sh/install.ps1 | iex"

# Or via npm (any platform)
npm install -g bun
```

Verify the install:
```bash
bun --version
```

### Install dependencies

```bash
npm install
```

### Development

Run the local dev server with hot reload:
```bash
npm run dev
# → http://localhost:3000
```

### Build distributable binaries

This single command builds the static site and compiles binaries for all four platforms:

```bash
npm run package
```

Output in `dist/`:
```
dist/
  out/                          ← static site assets (shared across all platforms)
  designer-faq-linux
  designer-faq-mac-intel
  designer-faq-mac-arm
  designer-faq-windows.exe
```

Zip the `dist/` folder and share it with your designers. Each person picks the binary for their OS and keeps it alongside the `out/` folder.

### Updating FAQ content

Edit `public/faq-data.json` to add, update, or remove questions, then run `npm run package` again to produce a new `dist/`.

The JSON structure:
```json
{
  "metadata": { "title": "...", "lastUpdated": "YYYY-MM-DD" },
  "categories": [
    {
      "id": "category-id",
      "title": "Category Title",
      "description": "Short description",
      "questions": [
        {
          "id": "question-id",
          "question": "The question?",
          "answer": "The answer. Supports **bold**, *italic*, and `code`."
        }
      ]
    }
  ]
}
```

### Project structure

```
designer-faq/
├── pages/
│   ├── _app.js          ← global layout and styles
│   └── index.js         ← main FAQ page
├── public/
│   └── faq-data.json    ← all FAQ content lives here
├── styles/
│   └── globals.css      ← all styling
├── serve.ts             ← Bun HTTP server (compiled into the binary)
├── next.config.js       ← Next.js config (output: export)
└── dist/                ← generated by `npm run package`, not committed
```
